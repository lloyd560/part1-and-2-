For this project, I built a Console app and a WPF app for course registration. Both versions had to follow the rules, check user input, and show correct results. The goal was to show that I can fix code, add features, and write C# code on my own. I did well at finding errors in the starter code and cleaning up the logic. My WPF design was simple and easy for users to understand. I made sure the buttons, lists, and messages were clear and helped prevent mistakes. The Console app used text and typing, while the WPF app was visual and user‑friendly. The WPF version used lists, buttons, and labels to make the process easier for the user. When debugging, I tested small parts of the program one step at a time. I used breakpoints and printed values to find problems. This helped me fix issues quickly and keep the code working. A challenge was changing the console logic into a WPF design. I had to rethink how the program worked with events and buttons. Solving this helped me learn more about C#, WPF, and building user‑friendly apps.
